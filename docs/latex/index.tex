{\bfseries{\doxylink{class_quick_tween_director}{Quick\+Tween\+Director}}} is a visual timeline editor and runtime playback system built on top of the {\bfseries{Quick\+Tween}} plugin. It lets you design multi-\/track tween animations directly inside the Blueprint editor, store them as reusable data assets, and play them back at runtime with just a couple of Blueprint nodes — no code required.

Think of it as a lightweight in-\/editor Sequencer that lives inside your Blueprint. You place colored step blocks on component tracks, arrange their timing visually, save the result as an asset, and then drive it all from a single {\ttfamily Create Director Player → Play} call at runtime.

\begin{quote}
{\bfseries{Requires\+:}} The {\bfseries{Quick\+Tween}} plugin must be installed and enabled in your project alongside this one. \end{quote}

\hypertarget{index_autotoc_md2}{}\doxysubsection{\texorpdfstring{Why Quick\+Tween\+Director?}{Why Quick\+Tween\+Director?}}\label{index_autotoc_md2}
If you have ever built a complex animation using Quick\+Tween alone, you know the problem. You end up with a wall of Begin\+Play nodes, magic delay numbers, and timing logic spread across multiple event graphs. When something needs to change, you have to hunt through all of it.

\doxylink{class_quick_tween_director}{Quick\+Tween\+Director} solves this by giving you a proper timeline. You can see all your component animations at once, drag steps around to get the timing right, and when you are done you have a clean asset that can be reused across multiple Blueprints.

Here is what it makes easy\+:


\begin{DoxyItemize}
\item Designing animations visually instead of guessing delay values
\item Running multiple component animations in parallel without manually coordinating them
\item Storing animations as data assets that serialize with your project
\item Reusing the same animation asset from any Blueprint that has matching components
\item Reacting to playback events — globally or per individual step
\item Exporting and importing animations as JSON for version control or sharing
\item Nesting Director animations inside larger Quick\+Tween sequences
\end{DoxyItemize}

The Director editor is embedded inside the {\bfseries{Blueprint editor}}. It is not a standalone window or Content Browser tool.


\begin{DoxyEnumerate}
\item Open any {\bfseries{Actor Blueprint}} by double-\/clicking it in the Content Browser.
\item In the Blueprint editor toolbar, click the {\bfseries{Director}} button.
\item The {\bfseries{Quick\+Tween Director}} panel opens as a dockable tab. Drag it wherever is comfortable — below the viewport, next to the event graph, or as a floating window.
\end{DoxyEnumerate}

The panel tracks the last-\/focused Actor Blueprint automatically. If you have multiple Blueprint editors open, clicking {\bfseries{Director}} from the right one switches the panel\textquotesingle{}s context.

\hypertarget{index_autotoc_md14}{}\doxysubsubsection{\texorpdfstring{Adding Tracks}{Adding Tracks}}\label{index_autotoc_md14}
Each {\bfseries{track}} targets one component from your Blueprint\textquotesingle{}s component hierarchy. Tracks run in parallel at runtime — steps on different tracks can overlap freely.

{\bfseries{To add a track\+:}}
\begin{DoxyEnumerate}
\item Click {\bfseries{+ Add Track}} in the toolbar.
\item The {\bfseries{Select Component}} dialog shows the component tree of your Blueprint.
\item Select a component and click {\bfseries{OK}}.
\end{DoxyEnumerate}

\begin{quote}
Only Blueprint SCS components (the ones you added yourself) appear in the picker. Inherited C++ native components can still be animated by calling {\ttfamily Bind Slot} manually at runtime. \end{quote}


{\bfseries{To reorder tracks\+:}} Use the {\bfseries{▲ / ▼}} arrow buttons on the left edge of each track label. The change is undoable with Ctrl+Z.

{\bfseries{To delete a track (and all its steps)\+:}} Click the {\bfseries{✕}} button on the track label. This is undoable.

\DoxyHorRuler{0}
\hypertarget{index_autotoc_md16}{}\doxysubsubsection{\texorpdfstring{Adding Steps}{Adding Steps}}\label{index_autotoc_md16}
A {\bfseries{step}} is a colored block on a track that represents one tween animation. Steps on the same track play sequentially relative to each other based on their start times, but steps across different tracks always play in parallel.

{\bfseries{To add a step\+:}}
\begin{DoxyEnumerate}
\item {\bfseries{Right-\/click}} on an empty area of a track row.
\item The {\bfseries{Add Step}} submenu appears, filtered to step types that make sense for that component class.
\item Select a type. The {\bfseries{Edit Step}} dialog opens so you can configure values and timing.
\item Click {\bfseries{Apply}}. The step appears on the track as a colored block.
\end{DoxyEnumerate}

The start time of the new step is placed at the position where you right-\/clicked, snapping to 0.\+05 s if Snap is enabled.

\hypertarget{index_autotoc_md36}{}\doxysubsubsection{\texorpdfstring{Exporting to JSON}{Exporting to JSON}}\label{index_autotoc_md36}
Click {\bfseries{Export JSON}} in the toolbar to write the entire asset to a {\ttfamily .json} file. The export is {\bfseries{lossless}} — it contains every field needed to fully restore the asset\+:


\begin{DoxyItemize}
\item Asset name, loop count, total duration
\item All tracks (GUID, label, component variable name)
\item All steps (GUID, track GUID, label, type, start time, duration, time scale, loops, loop type, easing, ease curve path, slot name, parameter name, all type-\/specific values, custom color)
\end{DoxyItemize}

{\bfseries{When to use this\+:}}
\begin{DoxyItemize}
\item Diffing animation changes in source control
\item Sharing animations between projects
\item Backing up before a large restructure
\end{DoxyItemize}

\begin{quote}
The JSON file is not used at runtime. It is an editor-\/only exchange format. \end{quote}

\hypertarget{index_autotoc_md100}{}\doxysubsection{\texorpdfstring{Tips for Juniors}{Tips for Juniors}}\label{index_autotoc_md100}
A few things that are easy to miss when getting started\+:


\begin{DoxyItemize}
\item {\bfseries{Nothing plays in editor.}} The timeline is for authoring only. To see the result, you must run PIE with {\ttfamily Create Director Player → Play} in Begin\+Play.
\item {\bfseries{The Duration badge is read-\/only.}} It reflects the furthest step end time. To make the animation longer, add a step or extend an existing one further out.
\item {\bfseries{Snap makes life easier.}} Turn it on early and keep it on. It keeps steps aligned to clean boundaries and makes timing predictable.
\item {\bfseries{Export before big changes.}} Use Export JSON as a backup before reorganizing tracks or doing large edits. You can always import it back.
\item {\bfseries{Check Binding Errors in PIE.}} If something is not animating, the first thing to check is the {\ttfamily Binding Errors} array on the player. It will tell you exactly which slots failed and why.
\item {\bfseries{Infinite loops are not allowed inside a Director.}} If a step needs to loop forever, you cannot do it inside the timeline. Instead, use a loop count of 1 in the step and handle repeating at the Director level, or use a regular Quick\+Tween alongside the Director.
\item {\bfseries{Material steps always need a MID.}} You cannot animate a material parameter directly on a Static Mesh component. You must create a {\ttfamily UMaterial\+Instance\+Dynamic}, apply it to the mesh, and bind it via {\ttfamily Bind Slot} before {\ttfamily Play}.
\end{DoxyItemize}
